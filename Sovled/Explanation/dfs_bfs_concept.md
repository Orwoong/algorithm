# DFS/BFS 개념 정리 (입문용)

## 1) DFS(Depth-First Search, 깊이 우선 탐색)

정의:
- 한 경로로 "끝까지" 내려간 뒤,
- 더 갈 곳이 없으면 이전 분기점으로 돌아와 다른 길을 탐색

구현 방식:
- 재귀 함수(가장 직관적)
- 또는 스택(stack) 직접 사용

핵심 자료구조:
- `visited` 배열(중복 방문 방지)
- 재귀 콜스택 또는 명시적 스택

간단한 사고 방식:
- "지금 노드에서 갈 수 있으면 계속 들어간다."
- "막히면 되돌아와서 다른 이웃을 본다."

---

## 2) BFS(Breadth-First Search, 너비 우선 탐색)

정의:
- 시작점에서 가까운 노드부터 차례대로 탐색
- 같은 거리(레벨)의 노드를 먼저 다 처리하고 다음 레벨로 진행

구현 방식:
- 큐(queue) 사용이 사실상 표준

핵심 자료구조:
- `visited` 배열
- `deque` 큐

간단한 사고 방식:
- "먼저 들어온 노드를 먼저 꺼낸다(FIFO)."
- "현재 노드의 이웃을 큐 뒤에 붙여서 레벨 순서로 진행한다."

---

## 3) DFS vs BFS 차이 한눈에

| 항목 | DFS | BFS |
|---|---|---|
| 탐색 방향 | 깊게 우선 | 넓게 우선 |
| 주요 자료구조 | 재귀/스택 | 큐 |
| 구현 난이도(입문) | 재귀는 쉬움, 흐름 추적은 어려울 수 있음 | 코드 구조는 단순, 큐 상태 추적 필요 |
| 최단 거리 보장(무가중치 그래프) | 보장 안 함 | 보장함 |
| 자주 쓰는 경우 | 연결 요소, 사이클 탐지, 백트래킹 | 최단 거리, 레벨 탐색 |

---

## 4) 시간복잡도와 공간복잡도

그래프를 인접 리스트로 표현하면:

- 시간복잡도(DFS/BFS 공통): `O(V + E)`
  - `V`: 정점 수
  - `E`: 간선 수
- 공간복잡도(DFS/BFS 공통): `O(V)`
  - `visited` 배열
  - DFS는 재귀 스택, BFS는 큐가 추가로 필요

---

## 5) visited가 왜 중요한가

그래프에는 사이클이 있을 수 있다.

예:
- `1 -> 2 -> 1` 같은 구조에서
- `visited` 없이 탐색하면 무한 반복 가능성이 생긴다.

그래서 규칙은 단순하다:
1. 노드를 처음 방문할 때 `visited[node] = True`
2. 이미 방문한 노드는 다시 탐색하지 않는다.

## 6) 백준 2606에 적용하면

문제 핵심:
- `1번 컴퓨터`에서 시작해 연결된 컴퓨터를 모두 찾는다.
- 시작점 제외한 방문 수를 출력한다.

DFS 방식:
- 재귀로 깊게 탐색하면서 새로 방문할 때마다 카운트 증가

BFS 방식:
- 큐로 레벨 순서 탐색하면서 새로 방문할 때마다 카운트 증가

두 방식 모두 정답은 동일하다.

---

## 7) 입문자가 자주 하는 실수

1. 양방향 간선을 한쪽만 추가하는 실수
   - `graph[a].append(b)`만 하고 `graph[b].append(a)`를 빼먹음
2. 방문 체크 시점이 늦는 실수
   - BFS에서 큐에 넣을 때 방문 처리를 안 하면 중복 삽입 가능
3. 시작 노드 카운트 처리 실수
   - 2606은 `1번` 자체는 제외해야 함
4. 재귀 종료 조건을 따로 안 써서 불안해하는 경우
   - DFS는 "미방문 이웃이 없으면 for문이 끝나며 자연 복귀"한다
